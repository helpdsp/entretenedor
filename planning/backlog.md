# Backlog base

## Planned stories

- **Como** usuario nuevo **quiero** registrarme mediante invitación o lista de espera **para** acceder a un entorno de aprendizaje exclusivo y controlado.
- **RF:** RF-01
- **AC:**
- [ ] El registro bloquea si no hay un `invite_token` válido en la URL o sesión.
- [ ] Formulario de waitlist (nombre, email) inserta en tabla `waitlist` con status `pending`.
- [ ] Platform Admin puede invocar la Edge Function `waitlist-invite` para disparar email de invitación (Resend).
- [ ] Al registrarse, el sistema ejecuta `domain_matching` contra la tabla `organizations`.
- [ ] Si hay match y la org tiene auto-join activo, el usuario se vincula automáticamente en `org_members`.
- **Como** usuario registrado **quiero** iniciar sesión con email y contraseña **para** acceder a mi contenido y progreso de forma segura.
- **RF:** RF-02
- **AC:**
- [ ] Login email/password via Supabase Auth (GoTrue).
- [ ] Persistencia de sesión con Refresh Token (manejo automático por Supabase client).
- [ ] Recuperación de contraseña funcional (flujo de email).
- [ ] Role-based access control (RBAC) verificado en el AppShell mediante el campo `profiles.role`.
- **Como** visitante curioso **quiero** ver una landing con la propuesta de valor **para** entender si FastTrack AI es la solución para mi equipo.
- **RF:** RF-03
- **AC:**
- [ ] Hero section con narrativa de "Atomic Learning".
- [ ] Sección de beneficios segmentada por Creador, Aprendiz y Org Admin.
- [ ] Footer con ThemeSwitcher (skins) y LanguageSwitcher (i18n).
- [ ] Tiempo de carga (LCP) < 1.5s en desktop.
- **Como** learner **quiero** que mi progreso en un átomo se reconozca en todas las rutas **para** no repetir nunca contenido que ya domino.
- **RF:** RF-04
- **AC:**
- [ ] Al cumplir la condición del player, se invoca el RPC `mark_atom_complete`.
- [ ] La tabla `user_progress` registra el completado con un UNIQUE constraint `(user_id, atom_id)`.
- [ ] El sistema detecta si el átomo ya estaba completado (`already_completed: true`) y no duplica créditos.
- [ ] Si el átomo pertenece a múltiples células o tracks, el estado visual se actualiza en todos instantáneamente.
- **Como** learner **quiero** ver la ruta de aprendizaje como un grafo visual **para** entender la estructura del conocimiento y mi posición en ella.
- **RF:** RF-08
- **AC:**
- [ ] Grafo renderizado con React Flow.
- [ ] Nodos con estados visuales claros: Not Started (gris), In Progress (borde gradiente), Completed (violeta/magenta con check icon).
- [ ] Atoms agrupados visualmente en "Cell Groups" (células).
- [ ] Nodos con `breaking_change` muestran un halo ámbar pulsante (RN-11).
- [ ] Zoom-to-fit automático al cargar el track.
- **Como** learner **quiero** consumir contenido a través de reproductores específicos por tipo **para** tener una experiencia de aprendizaje variada y efectiva.
- **RF:** RF-09
- **AC:**
- [ ] **QuizPlayer**: Evaluación de 4 opciones con feedback inmediato y score dinámico.
- [ ] **FlashcardPlayer**: Mazos con flip 3D (Framer Motion) y navegación fluida.
- [ ] **PlaybookPlayer**: Markdown enriquecido con checklist de lectura H2/H3.
- [ ] **VideoPlayer**: Mark complete automático al alcanzar el 80% del metraje.
- [ ] **TaskPlayer**: Sandbox HTML con checkbox de "I have completed this task".
- **Como** administrador de organización **quiero** asignar tracks a mi equipo y pagarlos de forma transparente **para** escalar el aprendizaje en mi empresa.
- **RF:** RF-12
- **AC:**
- [ ] Panel de gestión de miembros con invitaciones por email funcionales.
- [ ] Modal `AssignTrackModal` permite seleccionar un miembro/equipo y un track.
- [ ] Al confirmar, dispara la Edge Function `stripe-checkout` para pago pay-per-seat-per-track.
- [ ] Tras el éxito en Stripe (`stripe-webhook`), la asignación se marca como `active` en la DB.
- [ ] El miembro recibe una notificación realtime de "Track Asignado".
- **Como** creador **quiero** transformar un PDF en un track publicado en minutos **para** maximizar mi productividad y democratizar mi conocimiento.
- **RF:** RF-13
- **AC:**
- [ ] Paso 1 (Upload): Soporte para PDF y DOCX con drag & drop.
- [ ] Paso 2 (Fragmentation): Estrucutra de células/átomos sugerida por IA editable con dnd-kit.
- [ ] Paso 4 (Reutilization): Motor de búsqueda de similitud (pg_trgm) evita duplicar contenido existente (ADR-03).
- [ ] Paso 6 (Copilot): Chat lateral interactivo para refinar el contenido de los átomos generados.
- [ ] Persistencia de borrador automática (local + DB) para retomar en cualquier momento.
